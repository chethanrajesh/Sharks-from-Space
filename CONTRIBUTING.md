# Contributing

Thanks for your interest in contributing to Sharks From Space! Please follow these guidelines to make the process smooth and efficient.

## How to contribute

1. Fork the repository and create a branch named `feature/your-feature` or `bugfix/issue-123`.
2. Implement your changes and include tests where appropriate.
3. Commit with clear, concise messages.
4. Push to your fork and open a Pull Request (PR) against `main`.

## PR Checklist

- [ ] Describe the goal of the change and why it is needed.
- [ ] Add or update tests when applicable.
- [ ] Update `README.md` or documentation if behavior changes.
- [ ] Ensure code follows the style in the repo (PEP8 for Python).

## Coding conventions

- Use `black` (or follow PEP8) for formatting.
- Keep notebook outputs cleared before committing large changes to `.ipynb` files.
- Use descriptive variable and function names.

## Reporting issues

Open an issue with a minimal reproducible example, expected behavior, and actual behavior. Attach logs or screenshots if helpful.

## Pull request review

Expect maintainers to review your PR and request changes. Address comments promptly and keep PRs focused and small where possible.

Thanks — your contributions help make this project better for everyone! ❤️
